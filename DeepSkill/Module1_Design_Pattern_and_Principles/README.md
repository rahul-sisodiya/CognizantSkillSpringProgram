# Module 1 - Design Patterns and Principles

## Overview

This module focuses on the implementation of commonly used Software Design Patterns and Principles using Java. These design patterns help improve code reusability, maintainability, scalability, and flexibility in software applications.

## Objectives

* Understand Creational, Structural, and Behavioral Design Patterns.
* Implement design patterns using Java.
* Learn real-world applications of design patterns.
* Develop modular and maintainable software solutions.

## Exercises Completed

### Creational Design Patterns

1. Singleton Pattern

    * Implemented a Logger class ensuring only one instance exists throughout the application.

2. Factory Method Pattern

    * Implemented a document management system supporting Word, PDF, and Excel documents.

3. Builder Pattern

    * Implemented a Computer object with configurable components using the Builder Pattern.

---

### Structural Design Patterns

4. Adapter Pattern

    * Integrated multiple payment gateways through a common interface.

5. Decorator Pattern

    * Added notification channels dynamically using decorators.

6. Proxy Pattern

    * Implemented lazy loading and caching for image viewing.

---

### Behavioral Design Patterns

7. Observer Pattern

    * Implemented a stock market monitoring system with multiple observers.

8. Strategy Pattern

    * Implemented runtime selection of payment methods.

9. Command Pattern

    * Implemented a home automation system using command objects.

---

### Architectural Patterns

10. MVC Pattern

    * Implemented a student management application using the Model-View-Controller architecture.

11. Dependency Injection

    * Implemented constructor-based dependency injection for customer management services.

---
